Usabilidad animaciones

Las animaciones son usadas para como su nombre indica, animar cualquiera de los elementos del árbol DOM que se le asigne dicha animación.

Ejemplo:

@keyframes identifier {
0% { top: 0; }
50% { top: 30px; left: 20px; }
50% { top: 10px; }
100% { top: 0; }
}

Declaración de animaciones

Las animaciones se declaran mediante la propiedad @keyframes y mediante llaves({}), se engloban las distintas propiedades.
Hay dos formas de declarar los @keyframes:

1. Dentro de las llaves se colocan las propiedad “from” y “to”. De forma que (From) englobará las acciones que se realizaran primero. Y la propiedad (to) englobará las acciones que se realizaran al final. Osea después del from.

Ejemplo:

@keyframes identifier {
from { top: 0; }
to { top: 30px; left: 20px; }
}

2. Dentro de las llaves se colocan porcentajes, indicando la cantidad proporcinal de la animación, desde 0%(inicio) a 100%(fin).

Ejemplo:

@keyframes identifier {
0% { top: 0; }
50% { top: 20px; left: 20px; }
100% { top: 40px; left: 40px; }
}

Propiedades

Son aquellas que aplican comportamiento de la animación que se invoca.
Hemos visto anteriormente que la animación se puede invocar indicando todos los valores seguidos.
-Por ejemplo así:

div{
animation: pulsacion 5s infinite ease-out;
}

Pero las propiedades nos permiten dividir la invocación de la animación en partes, ya que cada propiedad se encargará de una función.


propiedades:

div{
animation-name: pulsacion;
animation-duration: 5s;
animation-iteration-count: infinite;
animation-timing-function: ease;
...
}

En el ejemplo he puesto las propiedades mas usadas. Pero mas adelante explocaré otras mas, que nos pueden ser útiles.

Propiedad animation-name

Es necesario usarla para aplicar la animación que debe estar declarada(creada) previamente.
Como su nombre indica admite el nombre de la animación. Tenemos que tener en cuenta que solo con el nombre no nos funcionará la animación. Por lo que tendrás que leer las siguientes propiedades para poder invocar combinarlas.

Ejemplo:

div{ animation-name: pulsacion; }
@keyframes pulsacion {
0% { width: 50px; height:50px;}
50% { width: 100px; height:100px; }
100% { width: 50px; height:50px; }
}

Propiedad animation-duration

Es usada para indicar la duración de la animación. Se expresa en segundos.Tenemos que tener en cuenta que solo con la duración no nos funcionará la animación. Por lo que tendrás que leer las siguientes propiedades para poder invocar combinarlas.

Ejemplo:

div{
animation-duration:1s;
}

Propiedad animation-timing-function

Esta propiedad especifica cómo una animación CSS debe avanzar sobre la duración de cada ciclo.
Una timing function definida dentro de un keyframe afecta a ese keyframe. Si no está definida para el keyframe, se aplica la timing function para la animación en general.

Los valores que puede tomar son:

animation-timing-function: ease; // La animación tiene un inicio lento, luego rápido, luego termina lentamente
ease

animation-timing-function: ease-in; // La animación tiene un inicio lento
ease-in

animation-timing-function: ease-out; // La animación tiene un final lento
ease-out

animation-timing-function: ease-in-out; // La animación tiene un inicio lento y un final lento
ease-in-out

animation-timing-function: linear; //La animación tiene la misma velocidad desde el inicio hasta el final
linear

animation-timing-function: step-start; // La animación se para al principio
step-start

animation-timing-function: step-end; // La animación se para al final
step-end

Propiedad animation-delay

Esta propiedad especifica el tiempo de retardo que debe transcurrir antes de comenzar la animación. Esto permite que la secuencia de animación no comience nada más ser aplicada a un elemento.
El valor 0, que es el valor por defecto, indica que la animación debería comenzar inmediatamente. Usando otro valor, definimos el retardo entre el momento en que el elemento recibe la animación
y el comienzo de la secuencia de la misma.
Si especificamos un valor negativo, la animación comenzará inmediatamente, pero no desde el principio de la misma, sino desde el punto que le hemos indicado.
Por ejemplo, si ponemos un retardo de -1s, la animación se iniciará inmediatamente pero comenzará en el segundo 1 de la secuencia.

Ejemplo:

div{
animation-delay: 2s;
}
@keyframes mymove {
from {left: 0px;}
to {left: 200px;}
}

Propiedad animation-iteration-count

Esta propiedad define el numero de veces que un ciclo de animación debe ser ejecutado antes de detenerse.

Ejemplos:

div{
animation-iteration-count: 3;
}

div{
animation-iteration-count: infinite;
}
@keyframes mymove {
from {left: 0px;}
to {left: 200px;}
}

Propiedad animation-direction

La propiedad animation-direction indica si la animación debe retroceder hasta el fotograma inicial al finalizar la secuencia, o si debe comenzar desde el principio al llegar al final.

Los valores que puede tomar son:

div{
 animation-direction: normal ⇨ Dirección normal de la animación creada.
}
NORMAL
div{
 animation-direction: reverse ⇨ La dirección opuesta a la dirección de la animación creada.
}
REVERSE
div{
  animation-direction: alternate ⇨ Alterna la dirección normal con la dirección opuesta.
}
ALTERNATE
div{
  animation-direction: alternate-reverse ⇨ Alterna la dirección normal con la dirección opuesta empezando por la opuesta.
}

Propiedad animation-fill-mode

Esta propiedad especifica el modo de cómo se comportará el CSS en función de la animación.
Por ejemplo la propiedad, 'fowards' especifica que, al finalizar la animación, el objeto debe tomar los valores de 'To' del Keyframe.
Del mismo modo que 'backwards' debe aplicarse con la propiedad 'delay'. Esto hace que durante el tiempo que la animación esté en delay, el objeto mantenga las propiedades CSS de 'From' del Keyframe.

Los valores que puede tomar son:

div{
 animation-fill-mode: none ⇨ El estilo se mantiene tal cómo debe de ser.
}
none
div{
 animation-direction: animation-fill-mode: forwards ⇨ Al finalizar la animación mantiene el estilo final ('To').
}
forwards
div{
 animation-delay: 5s; ⇨ Debe aplicarse con animation-delay
 animation-fill-mode: backwards ⇨ Mantiene el estilo inicial ('From') durante el 'delay'. Al finalizar vuelve al estado inicial.
}
backwards
div{
 animation-delay: 5s; ⇨ Debe aplicarse con animation-delay
  animation-fill-mode: both ⇨ Combina la propiedad forwards y backwards.
}
both

@keyframes slideout {
 from {
  margin-left: 0%;
  width: 100%;
 }
 to {
  margin-left: 88%;
  width: 150px;
 }
}

Propiedad animation-play-state

Play-State determina si una animación está en ejecución o en pausa. Puede ser consultada para determinar si la animación se está ejecutando. Además, su valor se puede establecer para pausar y reanudar una animación.
Reanudando una animación pausada, ésta empezará en el punto en el que fue pausada, en vez de empezar desde el principio.

Los valores que puede tomar son:

div{
 animation-play-state: running;
}
runing
div{
 animation-play-state: paused;
}
paused

@keyframes slideout {
 from {
  margin-left: 0%;
  width: 100%;
 }
 to {
  margin-left: 88%;
  width: 150px;
 }
}

Propiedad animation

Es una propiedad para invocar animaciones declaradas previamente.
Los valores que admite (animation) son:
*animation: <nombre> <duración(segundos)> (opcional)<infinite> (opcional)<velocidad>;
(Se le pueden añadir muchos mas valores pero yo he indicado los mas usados)

Ejemplo:

div{ animation: mymove 5s infinite ease-out; }
@keyframes mymove {
from {left: 0px;}
to {left: 200px;}
}
